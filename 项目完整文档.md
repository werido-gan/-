## 项目概述

### 项目简介
智能物流看板是一个基于前后端分离架构的企业级物流管理系统，提供订单管理、部门管理、物流信息查询、预警监控等功能，帮助企业实现物流信息的集中管理和实时监控。

### 核心特性
- **实时物流追踪**：集成第三方物流API，实时获取物流信息
- **智能状态识别**：自动识别物流状态，包括异常件状态
- **预警系统**：延迟发货预警、运输异常预警
- **批量操作**：支持批量导入、导出、刷新物流信息
- **多维度统计**：按部门、客户、时间等多维度统计分析
- **权限管理**：基于JWT的用户认证和权限控制
- **操作审计**：完整的操作日志记录

## 数据库设计

### 数据库表结构

#### 1. orders（订单表）
```sql
CREATE TABLE orders (
  id INT PRIMARY KEY AUTO_INCREMENT,
  order_number VARCHAR(255) NOT NULL UNIQUE COMMENT '物流单号',
  customer_name VARCHAR(255) NOT NULL COMMENT '客户/项目名称',
  department_key VARCHAR(50) NOT NULL COMMENT '业务部门',
  user_id INT COMMENT '创建用户ID',
  carrier VARCHAR(100) COMMENT '承运商名称',
  carrier_code VARCHAR(50) COMMENT '承运商代码',
  status ENUM('pending', 'in_transit', 'delivered', 'returned', 'exception') 
    DEFAULT 'pending' COMMENT '订单状态',
  warning_status ENUM('none', 'delay_shipment', 'transit_abnormal') 
    DEFAULT 'none' COMMENT '预警状态',
  is_archived BOOLEAN DEFAULT FALSE COMMENT '逻辑删除标记',
  details JSON COMMENT '订单详情（JSON格式）',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  INDEX idx_order_number (order_number),
  INDEX idx_department_key (department_key),
  INDEX idx_status (status),
  INDEX idx_warning_status (warning_status),
  INDEX idx_is_archived (is_archived),
  INDEX idx_created_at (created_at),
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

#### 2. users（用户表）
```sql
CREATE TABLE users (
  id INT PRIMARY KEY AUTO_INCREMENT,
  username VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
  password VARCHAR(255) NOT NULL COMMENT '密码（加密）',
  email VARCHAR(255) COMMENT '邮箱',
  department VARCHAR(50) COMMENT '所属部门',
  phone VARCHAR(20) COMMENT '电话',
  role ENUM('admin', 'user') DEFAULT 'user' COMMENT '角色',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  last_login TIMESTAMP COMMENT '最后登录时间',
  INDEX idx_username (username),
  INDEX idx_role (role)
);
```

#### 3. departments（部门表）
```sql
CREATE TABLE departments (
  id INT PRIMARY KEY AUTO_INCREMENT,
  `key` VARCHAR(50) NOT NULL UNIQUE COMMENT '部门唯一标识',
  name VARCHAR(100) NOT NULL COMMENT '部门名称',
  description TEXT COMMENT '部门描述',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  INDEX idx_key (`key`)
);
```

#### 4. operation_logs（操作日志表）
```sql
CREATE TABLE operation_logs (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT COMMENT '用户ID',
  username VARCHAR(50) COMMENT '用户名',
  operation_type ENUM('import', 'export', 'delete', 'archive', 'restore', 'update', 'create', 'login', 'logout') 
    NOT NULL COMMENT '操作类型',
  target_type ENUM('order', 'user', 'department', 'system') COMMENT '目标类型',
  target_id VARCHAR(255) COMMENT '目标ID',
  details JSON COMMENT '操作详情',
  ip_address VARCHAR(45) COMMENT 'IP地址',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  INDEX idx_user_id (user_id),
  INDEX idx_operation_type (operation_type),
  INDEX idx_created_at (created_at)
);
```

#### 5. task_executions（任务执行表）
```sql
CREATE TABLE task_executions (
  id INT PRIMARY KEY AUTO_INCREMENT,
  task_name VARCHAR(255) NOT NULL COMMENT '任务名称',
  task_type VARCHAR(50) COMMENT '任务类型',
  status ENUM('pending', 'running', 'completed', 'failed') 
    DEFAULT 'pending' COMMENT '任务状态',
  progress INT DEFAULT 0 COMMENT '进度（0-100）',
  result JSON COMMENT '执行结果',
  error_message TEXT COMMENT '错误信息',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  INDEX idx_task_name (task_name),
  INDEX idx_status (status)
);
```

### 数据库索引优化

#### 主要索引
- **订单表**：单号、部门、状态、预警状态、归档标记、创建时间
- **用户表**：用户名、角色
- **部门表**：部门key
- **操作日志表**：用户ID、操作类型、创建时间
- **任务执行表**：任务名称、状态

#### 性能优化建议
1. 定期清理归档订单数据

3. 定期清理已完成的任务记录
